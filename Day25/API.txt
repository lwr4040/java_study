아래와 같은 그룹으로 클래스들을 분류한다.
 - 인터페이스 (interface)
 	다중상속용 클래스
 - 클래스 ( Class )
	일반 클래스와 추상클래스
 - 열거형 ( Enum )
	안배움.C언어의 잔재
	연속된 데이터들에 숫자값을 부여하기 위한 클래스
	ex) 도레미파솔라시도에 각각 0부터 8까지 부여
	상수화 변수로 대체가 가능	( pbulic static final )
 - 예외 ( Exception ) **
	외부 원인으로 인하여 실행이 안되는 문제
	코드상으로는 원인이 보이지않는다.
	ex)숫자 입력 코드
	처리 가능 , 예외 처리 단원에서 배움 ( try ~ catch )

 - 에러 ( Error )
 	100%실행이 안되는 문제
	ex) 코드 이탈 , 문법 오류

 - 주석형 ( Annotation )
	안배움.나중에 나옴 ( Jsp , Spring )
	@로 시작하는 키워드들을 정의한 클래스
	ex) @Override
	바로 아래 지역에 강제적인 효과를 부여

 [3] 클래스 상세	중앙 영역
 - 이 클래스에 대한 상제 정보가 표시
 - 값, 코드는 나오지 않고, 형태만 제시

 [ 필드의 개요 ]	변수 정보
 [ 생성자의 개요 ]	생성자 정보
 [ 메소드의 개요 ]	메소드 정보
